# Pult1.2 — правила для Claude Code

## Ветка — всегда `main`

**Коммитить и пушить напрямую в `main`.** Никаких автоматически создаваемых фича-веток типа `claude/<прилагательное-фамилия>` — это путает пользователя и его работа не видна на другом ноуте, пока кто-то не смержит.

Фича-ветку создавать **только если пользователь явно об этом попросил** (например «сделай в отдельной ветке», «через PR»). По умолчанию — прямо в `main`.

**Почему:** 2026-04-17 офисный Клод создал ветку `claude/upbeat-pasteur-0723b7` и запушил туда всю работу, не в `main`. Дома пользователь увидел устаревшую версию и потратил время на разбирательство. Такого быть не должно.

## Auto-push после правок

После завершения любой логической единицы изменений (фича, фикс, рефакторинг) — сразу:

```bash
git add -A
git commit -m "<тип: краткое описание>"
git push
```

Без напоминания от пользователя. Формат коммит-сообщения — как в существующей истории: `feat:`, `fix:`, `refactor:`, `chore:` и т.п.

**Почему:** пользователь работает на двух машинах (офисный ноут + домашний). Без дисциплины auto-push работа застревает локально, и на другом ноуте видна устаревшая версия.

**Когда НЕ пушить автоматически:**
- Пользователь явно остановил работу на середине («стой», «подожди», «не то делаем») — не коммитить черновое состояние.
- Есть незавершённая миграция/билд с ошибками — сначала починить, потом коммит.

**Перед началом сессии:** `git pull` в `main`, чтобы подтянуть правки с другого ноута.

**Если push упал** (нужен токен, конфликт, и т.п.) — сообщить пользователю, не молчать.
